# Agent UI Revamp Design

**背景**
- 现有 Agent 页面信息层级不清、留白偏大、主内容区不够聚焦。
- 事件流中出现长字符串（无空格 token / URL / JSON）时，存在“一行无限拉长”挤压左右信息的风险。

**目标**
- 采用更清晰的“双栏主舞台时间线”布局：左侧会话列表，右侧任务执行时间线，底部固定任务输入区。
- 任何消息/事件内容都不能把容器撑出横向，长内容必须自动换行或只在卡片内部滚动。
- 不改动 Agent 的后端协议与核心运行逻辑（SSE 事件结构、会话/Workspace 默认值规则保持不变）。

**非目标**
- 不新增复杂的 Inspector 三栏结构（可以后续再做）。
- 不引入新的 UI 框架或大规模样式系统改造（延续 Mantine）。

---

## 方案选择

**采用方案 A（推荐）：双栏主舞台时间线**
- 左栏：会话列表（新建、状态 badge、`...` 菜单：重命名/完成/删除）。
- 右栏顶部：当前会话标题 + 状态、Workspace 选择（账号级默认，切换立即保存）、刷新等工具按钮。
- 右栏中部：事件时间线（滚动区），空态引导明确。
- 右栏底部：固定 Composer（任务输入、附件、运行按钮），运行中状态明确、禁用原因明确。

---

## 信息架构与交互

**会话列表（左栏）**
- 标题/状态/菜单保持一行，不允许横向撑开（确保 `flex` 子项 `min-width: 0`，标题 `truncate`）。
- “新建会话标题”输入框只负责创建会话，不复用任务输入框。

**主舞台（右栏）**
- 顶部工具栏：
  - Workspace `Select`（显示为“默认 Workspace”，无 workspace 时显示引导）。
  - “刷新会话列表”放在工具栏右侧，避免分散注意力。
- 事件时间线：
  - 事件卡片统一视觉语义（text/tool_start/tool_result/error/done）。
  - tool 输入/输出默认折叠，展开后仍遵守“不可撑宽”规范。
- Composer（底部固定）：
  - 输入支持 Enter 发送（Shift+Enter 换行，如当前逻辑）。
  - 附件列表在 composer 内部展示，删除不影响布局。

---

## 长内容不撑宽（硬性规范）

对所有事件内容区域（包括 text、tool 输入、tool 输出、error）统一应用：
- `white-space: pre-wrap`（保留换行并可自动换行）
- `overflow-wrap: anywhere`（无空格长串也可断行）
- `word-break: break-word`（补充断行策略）
- 容器保证 `max-width: 100%`，避免横向溢出影响整列

对 `pre`/代码块：
- 默认同样允许断行（更适合日志/JSON 阅读）。
- 如果出现极端场景（超长单行二进制/大块无意义文本），允许仅在卡片内部出现横向滚动：
  - 在 `pre` 外再包一层 `Box`/`Paper`，设置 `overflowX: auto`，同时保持外层卡片不溢出。

---

## 文案与一致性

- 将页面关键文案统一为中文（如 “Agent Sessions”“New task...” 等）。
- 空态文案明确下一步动作：
  - 未选择会话：提示先选择或创建会话
  - 事件为空：提示在底部输入任务并运行
  - 无 workspace：提示去 Settings 创建 workspace

---

## 验收标准

1. 任何事件内容（含超长 URL/无空格 token/JSON）都不会导致页面出现横向撑宽挤压其它信息。
2. 右侧 Composer 固定在底部，事件时间线滚动时输入区不跳动。
3. 会话列表与主舞台的信息层级清晰，用户可以在 10 秒内完成：创建会话 -> 选择 workspace -> 运行任务。
4. 不引入重复的业务逻辑实现：继续复用现有 `agentStore`、`api`、settings 默认 workspace 逻辑。
